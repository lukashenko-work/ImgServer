Сброс счетчика id
Для сброса счетчика (последовательности) id в PostgreSQL, если таблица пуста или данные не важны, используйте
TRUNCATE TABLE table_name RESTART IDENTITY;
Если данные нужно сохранить, используйте
SELECT setval(pg_get_serial_sequence('table_name', 'id'), 1, false); для установки счетчика на 1. 

Логирование запросов в Postgres
ALTER SYSTEM SET log_statement = 'all';
SELECT pg_reload_conf();

имя

мс).

Классическая проблема прав доступа. Docker-демон работает от имени
root, а ваш текущий пользователь (ubuntu) не имеет прав на запись в его сокет.
Самый быстрый фикс (разово):
Просто добавьте sudo перед командой:
bash

sudo docker compose up -d --build


Правильный фикс (чтобы не писать sudo постоянно):
Добавьте своего пользователя в группу docker:

    Добавьте пользователя:
    bash

    sudo usermod -aG docker $USER

    Примените изменения:
    Чтобы группа обновилась без перезагрузки всей сессии, выполните:
    bash

    newgrp docker

    
    (Или просто выйдите из SSH и зайдите заново).
    Проверьте:
    Запустите команду снова без sudo:
    bash

    docker compose up -d --build


Если всё равно не пускает, проверьте права на сам сокет (крайняя мера):
bash

sudo chmod 666 /var/run/docker.sock
